## 📝 커밋 메시지 작성 규칙

### 기본 형식
```
태그: 제목
```
- 태그는 영어로 작성하되 첫 글자는 대문자
- `:` 뒤에만 공백 사용

### 커밋 태그 분류

#### 🚀 기능 관련
| 태그 | 설명 |
|------|------|
| `Feat` | 새로운 기능 추가 |
| `Fix` | 버그 수정 |
| `Design` | CSS/UI 변경 |

#### 🔧 개선 관련
| 태그 | 설명 |
|------|------|
| `Style` | 코드 포맷팅 (논리적 변경사항 없음) |
| `Refactor` | 코드 리팩토링 (기능/버그 수정 없이 구현 개선) |
| `Comment` | 주석 추가 및 변경 |

#### 📋 기타
| 태그 | 설명 |
|------|------|
| `Establish` | 초기 환경설정 / 설정 파일 추가 |
| `Docs` | 문서 수정 |
| `Test` | 테스트 코드 추가 |
| `Chore` | 빌드 업무, 패키지 매니저 수정 |
| `Remove` / `Rename` | 파일 삭제 / 파일명 변경 |
| `!HOTFIX` | 긴급한 버그 수정 (핫픽스) |
